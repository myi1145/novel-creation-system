# 01_基础卡槽导入导出方向确认

## 1. 文档目的

在候选方向对比基础上完成阶段拍板，统一“先做什么、暂不做什么”的团队口径，避免并行扩散到新卡型扩展与自动抽取方向。

## 2. 结论先行

下一阶段主方向确认为：**四类基础卡槽的 JSON / CSV 导入导出。**

本阶段定位为 docs-only 立项，不进入功能实现。

## 3. 主方向

主方向定义：

**围绕角色卡、术语卡、势力卡、地点卡，建立可审计、可校验、可回溯的 JSON / CSV 导入导出能力设计。**

该方向默认遵守：导入仅进入卡槽工作态，不等同 Canon 正史写入。

## 4. 第一批 MVP 方向

第一批 MVP 方向限定为：

1. JSON 导出；
2. JSON 导入；
3. CSV 导出；
4. CSV 导入；
5. 模板下载；
6. 导入校验报告。

## 5. 暂不采用方向

当前阶段明确暂不采用：

- Excel 导入导出；
- Markdown 导入；
- 从正文自动抽取；
- LLM 自动建卡；
- 批量写 Canon；
- 自动生成 ChangeSet。

## 6. 为什么现在做这个方向

1. 四类基础卡槽已完成最小人工维护闭环，进入“效率与迁移能力”补齐窗口；
2. 继续扩新卡型会扩大维护面，但不能解决现有卡槽高频的批量操作痛点；
3. JSON + CSV 可同时覆盖“系统级备份迁移”与“作者表格协作”；
4. docs-only 先行可先锁定边界，避免开发阶段引入 Canon / ChangeSet 绕行风险。

## 7. 与角色卡 / 术语卡 / 势力卡 / 地点卡的关系

- 本方向不新增卡型；
- 本方向仅提升既有四类卡槽的批量维护、备份与迁移效率；
- 本方向不改变四类卡槽字段中枢定义，不改变“是否进入正式设定”的门禁语义；
- 本方向不将导入结果自动升级为正式设定事实。

## 8. 一句话拍板

**下一阶段先让四类基础卡槽能够批量导入、导出、备份和迁移，但不把导入等同于进入正式设定。**
