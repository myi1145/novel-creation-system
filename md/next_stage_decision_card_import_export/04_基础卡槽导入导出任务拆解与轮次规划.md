# 04_基础卡槽导入导出任务拆解与轮次规划

## 1. 文档目的

给出基础卡槽导入导出在后续实现阶段的轮次化推进框架，确保“先审计、后实现、分轮验收、边界可控”。

## 2. 轮次规划结论先行

建议按五轮推进：

1. 第 0 轮：审计四类卡槽字段与现有 CRUD 接口；
2. 第 1 轮：设计 JSON / CSV 格式与模板；
3. 第 2 轮：补齐导出与模板下载；
4. 第 3 轮：补齐导入、校验与结果报告；
5. 第 4 轮：前端入口、测试、文档与阶段验收。

## 3. 推荐五轮推进

### 3.1 第 0 轮：审计四类卡槽字段与现有 CRUD 接口

- 目标：固化四类卡槽字段口径、主键规则、可复用接口边界；
- 不做：不开发新接口、不改数据库；
- 完成标志：形成字段映射表与不可破坏约束清单。

### 3.2 第 1 轮：设计 JSON / CSV 格式与模板

- 目标：确定 JSON/CSV 输入输出规范、模板结构、版本字段；
- 不做：不进入导入解析实现；
- 完成标志：格式说明与模板定义可供联调。

### 3.3 第 2 轮：补齐导出与模板下载

- 目标：实现四类卡槽导出与模板下载能力；
- 不做：不实现导入写入；
- 完成标志：导出文件可回读、模板可用于人工填充。

### 3.4 第 3 轮：补齐导入、校验与结果报告

- 目标：实现导入校验、最小重复策略、结构化导入报告；
- 不做：不做自动 merge、不做 Canon 入史；
- 完成标志：导入成功/失败/跳过可追踪，可定位问题行与字段。

### 3.5 第 4 轮：前端入口、测试、文档与阶段验收

- 目标：交付最小作者入口、测试集与阶段签收文档；
- 不做：不扩展 Excel / Markdown / 自动抽取；
- 完成标志：入口可用、测试通过、文档齐套并完成验收。

## 4. 每轮目标

每轮均要求满足以下共性目标：

1. 不破坏 Canon / ChangeSet 主链；
2. 输出物可被下一轮直接复用；
3. 每轮结束可独立验收并记录风险。

## 5. 每轮不做事项

跨轮统一不做事项：

- 不扩新卡型；
- 不做 Excel / Markdown；
- 不做正文自动抽取与 LLM 自动建卡；
- 不引入批量 Canon 写入与自动 ChangeSet 生成。

## 6. 每轮完成标志

- 第 0 轮：审计报告签收；
- 第 1 轮：格式规范评审通过；
- 第 2 轮：导出与模板下载联调通过；
- 第 3 轮：导入校验与报告回归通过；
- 第 4 轮：前端入口与文档验收通过。

## 7. 轮次依赖关系

- 第 0 轮是第 1~4 轮前提；
- 第 1 轮完成后，方可开展第 2/3 轮实现；
- 第 2 与第 3 轮完成后，方可开展第 4 轮联通验收；
- 不允许跳过前置轮次直接并行开发。

## 8. 不应出现的执行偏差

1. 未完成字段审计即启动导入实现；
2. 将导入功能误扩展为智能抽取项目；
3. 用“快捷路径”直接写 Canon 或伪造 ChangeSet；
4. 在 MVP 阶段承诺 Excel/Markdown 全支持；
5. 在 00~04 未评审通过前提前进入 05 执行任务单。

## 9. 与未来 05 执行任务单的关系

本文件用于规划，不是执行工单。仅当 00~04 决策链评审通过、并明确进入实现阶段后，才可编制 05 执行任务单。

## 10. 最终确认结论

当前阶段应先完成“基础卡槽导入导出”立项收口，并以 00~04 作为唯一上游输入。完成评审前，不进入实现开发。
