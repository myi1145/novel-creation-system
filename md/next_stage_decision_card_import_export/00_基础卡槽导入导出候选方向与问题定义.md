# 00_基础卡槽导入导出候选方向与问题定义

## 1. 文档目的

在四类基础卡槽（角色卡、术语卡、势力卡、地点卡）完成最小人工维护闭环与字段体验收口后，识别下一阶段最具收益的问题，并对导入导出相关候选方向进行对比，形成可执行的问题定义。

## 2. 当前已完成能力回顾

基于当前阶段交接与验收文档，以下能力已形成阶段性结论：

- 四类基础卡槽均已具备人工新建、编辑、查看的最小闭环；
- 四类卡槽字段口径已完成一轮作者体验收口；
- 当前卡槽定位明确为“人工维护入口”，未绕过 Canon / ChangeSet 主链；
- 当前阶段不应再扩展新卡型，应先提升已有四类卡槽的可维护效率与可迁移能力。

据此判断：**四类基础卡槽已经具备进入“导入导出立项”条件。**

## 3. 当前基础卡槽使用断点

尽管四类卡槽已可人工维护，但在规模化使用中存在以下断点：

1. 手工录入效率低，初始化与批量补录成本高；
2. 外部已有设定表（历史表格/协作文档）难以快速迁入；
3. 项目缺少标准化备份与迁移路径；
4. 跨项目复用成本高，重复录入风险大；
5. 批量导入错误缺少统一校验反馈，人工排错成本高。

**当前最需要解决的是：作者已经能手工维护四类基础卡槽，但批量迁移、备份、整理和初始化仍不方便。**

## 4. 候选方向对比

### 4.1 JSON 导入导出

- 优势：结构表达能力强，字段层级与对象语义完整，适合全量备份与跨项目迁移；
- 成本：作者直接手编门槛较高；
- 结论：应作为首批核心格式之一（机器友好与完整迁移优先）。

### 4.2 CSV 导入导出

- 优势：作者与编辑协作门槛低，易于批量整理与表格处理；
- 成本：层级表达受限，需要明确扁平字段映射；
- 结论：应作为首批核心格式之一（人工协作友好优先）。

### 4.3 Excel 导入导出

- 优势：作者习惯好，支持多 sheet；
- 成本：解析与格式兼容复杂度高，首轮实现风险大；
- 结论：首轮暂不优先，后置到 JSON/CSV 稳定后评估。

### 4.4 Markdown 导入

- 优势：自由文本友好；
- 成本：结构稳定性差，字段映射与一致性校验复杂；
- 结论：首轮暂不优先。

### 4.5 模板下载

- 优势：降低作者填表门槛，统一字段口径；
- 成本：需先固化字段与列映射；
- 结论：应纳入首批 MVP（与 CSV 配套）。

### 4.6 导入校验报告

- 优势：将失败原因结构化，降低排错成本，便于回改；
- 成本：需定义错误码、行级/字段级定位；
- 结论：应纳入首批 MVP。

### 4.7 重复卡片处理

- 优势：避免批量导入造成数据污染；
- 成本：需先明确同名判定与冲突策略；
- 结论：应纳入首批 MVP 的策略约束。

### 4.8 从正文自动抽取

- 优势：自动化程度高；
- 成本：识别准确性、可解释性、错误纠偏成本高；
- 结论：当前阶段不优先。

### 4.9 LLM 自动建卡

- 优势：可减少初始整理工作；
- 成本：事实漂移风险高，且易与 Canon 准入边界混淆；
- 结论：当前阶段不优先。

## 5. 本阶段建议优先方向

本阶段建议优先方向为：

1. 四类基础卡槽的 JSON 导入导出；
2. 四类基础卡槽的 CSV 导入导出；
3. CSV 模板下载；
4. 导入前字段校验与导入结果报告；
5. 重复卡片处理最小策略（先保守后扩展）。

优先理由：
- 直接解决当前高频使用断点；
- 在不扩新卡型的前提下显著提升作者维护效率；
- 与 Canon / ChangeSet 中枢边界兼容，不改变主链语义。

## 6. 本阶段暂不优先方向

本阶段暂不优先：

- Excel 导入导出；
- Markdown 导入；
- 从正文自动抽取；
- LLM 自动建卡；
- 批量写 Canon；
- 自动生成 ChangeSet。

边界理由：
- 当前阶段目标是“可控批量维护”，不是“智能抽取与自动入史”；
- 任何影响 Canon 的动作仍必须走既有 ChangeSet 准入链路。

## 7. 最终问题定义

下一阶段要解决的核心问题定义为：

**在不改变 Canon / ChangeSet 主链门禁的前提下，为四类基础卡槽建立 JSON/CSV 的批量导入导出、模板与校验报告能力，使作者能够完成初始化、备份、迁移与批量整理。**

配套澄清：

1. 当前不继续扩新卡型（如物品卡、伏笔卡），因为已有四类卡槽的可维护效率尚未达标；
2. 导入导出优先级高于新卡型扩展，因为其覆盖现有四类卡槽全部使用者且收益更直接；
3. 第一轮格式优先 JSON + CSV；
4. 导入行为不等同于进入 Canon，必须保持与 ChangeSet 的制度边界。
