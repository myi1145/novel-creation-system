# 02_基础卡槽导入导出阶段目标与边界

## 1. 文档目的

定义“基础卡槽导入导出”阶段的唯一目标、问题范围与严格边界，确保后续任务不偏移为接口实现或智能抽取项目。

## 2. 阶段定位

本阶段定位为：**基础卡槽导入导出立项阶段（docs-only）。**

阶段职责是固化方向与边界，不进入后端、前端、数据库实现。

## 3. 本阶段唯一目标

**建立四类基础卡槽的 JSON / CSV 导入导出与校验报告能力设计。**

## 4. 本阶段要解决的问题

1. 手动创建卡片效率低；
2. 外部表格设定无法快速迁入；
3. 卡槽无法标准化备份；
4. 卡槽无法跨项目迁移；
5. 导入错误缺少统一报告。

## 5. 本阶段严格边界

- 不做 Excel；
- 不做 Markdown 导入；
- 不做自动抽取；
- 不做 LLM 自动建卡；
- 不写 Canon；
- 不生成 ChangeSet；
- 不触发 stale；
- 不扩展新卡型；
- 不进入接口/数据库/UI 开发。

## 6. 允许的最小能力类型

在 docs-only 前提下，本阶段仅允许输出：

1. 四类卡槽导入导出的目标能力定义；
2. JSON/CSV 格式与字段映射建议；
3. 模板下载与导入校验报告口径；
4. 重复数据处理策略建议；
5. 与 Canon / ChangeSet 的边界约束；
6. 后续实现轮次建议与验收口径。

## 7. 后置事项

以下事项后置，不纳入本阶段：

- Excel / Markdown 格式支持；
- 正文自动抽取建卡；
- LLM 自动建卡与自动纠错；
- 复杂冲突策略（如自动 merge）；
- 批量 Canon 写入与自动 ChangeSet 生成。

## 8. 最终确认结论

当前阶段仅完成“基础卡槽导入导出”立项收口。阶段输出止于 00~04 决策文档链，不进入实现开发，不生成执行任务单。
