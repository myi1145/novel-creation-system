# 03_基础卡槽导入导出MVP范围

## 1. 文档目的

定义基础卡槽导入导出在后续实现阶段的最小可交付范围，确保 MVP 可落地、可验收、可审计。

## 2. MVP 结论先行

MVP 聚焦四类基础卡槽的标准文件批量维护能力，不扩展为智能抽取或自动入史系统。

## 3. MVP 必须覆盖

1. 角色卡 JSON 导入导出；
2. 术语卡 JSON 导入导出；
3. 势力卡 JSON 导入导出；
4. 地点卡 JSON 导入导出；
5. 四类卡槽 CSV 导入导出；
6. 四类卡槽 CSV 模板下载；
7. 导入前字段校验；
8. 导入结果报告；
9. 重复名称处理策略。

## 4. MVP 不包含

- Excel；
- Markdown；
- 自动抽取；
- LLM 自动建卡；
- 批量写 Canon；
- 自动生成 ChangeSet。

## 5. JSON 格式建议

建议统一顶层结构：

- `version`：模板版本；
- `card_type`：卡槽类型（character / terminology / faction / location）；
- `project_ref`：项目标识（用于导入校验）；
- `items`：卡片数组。

每条 `item` 建议包含：

- `name`（或术语名字段）；
- `type`；
- `summary` / `definition`；
- `first_appearance_chapter`；
- `status`；
- `source_note`。

原则：字段尽量与当前四类卡槽最小可维护字段对齐，禁止内嵌“自动入 Canon”语义字段。

## 6. CSV 字段建议

建议按“每类卡槽一份模板”输出，至少包含：

- 主标识字段（名称/术语名）；
- 类型字段；
- 核心说明字段（简介/定义）；
- 首次出现章节；
- 当前状态；
- 来源备注。

要求：

- 列名稳定且中英文映射固定；
- 空值规则、必填规则在模板头部说明；
- 同类卡槽模板字段顺序固定，便于批量处理。

## 7. 导入校验规则建议

导入前校验最小规则建议：

1. 文件级：格式合法、模板版本可识别；
2. 结构级：必填列存在；
3. 字段级：必填项非空、枚举字段取值合法、长度范围合法；
4. 语义级：同文件内主键重复检测；
5. 安全级：禁止携带 Canon 直写字段、禁止 ChangeSet 伪造字段。

## 8. 重复处理策略建议

推荐策略：

- 第一轮默认 `skip_duplicates`；
- 报告中返回重复项清单与跳过原因；
- 后续版本再评估 `overwrite` / `merge` 扩展策略。

## 9. 最小后端要求

后续若进入实现，最小后端要求为：

1. 四类卡槽导入导出接口能力；
2. 模板下载能力；
3. 校验报告结构化返回（文件级、行级、字段级）；
4. 导入写入仅作用于卡槽维护域，不触碰 Canon 写入链。

## 10. 最小前端要求

后续若进入实现，最小前端要求为：

1. 四类卡槽统一“导入 / 导出 / 下载模板”入口；
2. 导入参数最小配置（卡槽类型、重复策略）；
3. 导入结果报告可视化（成功/失败/跳过统计 + 明细）；
4. 明确边界提示：导入不等同正式设定入史。

## 11. 最小测试要求

后续若进入实现，最小测试要求为：

1. JSON/CSV 成功导入与导出回读一致性测试；
2. 模板字段完整性测试；
3. 必填缺失、类型非法、重复项场景测试；
4. 导入报告结构正确性测试；
5. Canon / ChangeSet 边界不被绕过测试。

## 12. 验收口径

满足以下条件可判定 MVP 验收通过：

- 四类卡槽可通过 JSON/CSV 完成批量导入导出；
- 模板下载可指导作者批量整理；
- 导入错误可被报告定位并回改；
- 重复项默认安全跳过；
- 未引入 Excel、自动抽取、LLM 自动建卡、Canon 直写与自动 ChangeSet。

**这一阶段的 MVP，不是做智能抽取，而是先让作者能用标准文件批量维护基础设定卡。**
