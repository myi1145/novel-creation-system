# 03_下游依赖失效与重跑治理MVP范围

## 1. 文档目的

本文件用于在立项层定义“下游依赖失效与重跑治理”最小可成立范围，为后续实现决策提供稳定约束。

本文件仅定义 MVP 范围，不定义实现任务。

## 2. MVP 结论先行

下游依赖失效与重跑治理的 MVP 在立项层应收敛为：

**先定义失效提示与人工确认重跑的最小闭环语义，不提前定义自动重算实现。**

## 3. MVP 必须回答的问题

立项层 MVP 至少要覆盖：
- 哪些上游改动会导致哪些下游对象或结果应提示“可能失效”；
- 失效提示与重跑引导的最小交互语义是什么；
- 哪些行为可进入后续实现讨论，哪些必须后置；
- 最小验收口径是什么（文档层验收）。

## 4. MVP 最小范围（立项语义）

### 4.1 失效对象范围

至少覆盖以下高频失效关系：
- `ChapterBlueprint` 修改后，`SceneCard` / `ChapterDraft` 可能失效；
- `SceneCard` 修改后，`ChapterDraft` 可能失效；
- `ChapterDraft` 修改后，Gate 评估结果与后续发布相关结论可能失效。

### 4.2 治理层级范围

- 第一层：失效提示（标记“可能过期”的下游对象或结果）；
- 第二层：人工确认重跑引导（明确建议重跑路径与顺序）；
- 第三层：自动治理仅作为后置议题，不纳入本阶段 MVP。

### 4.3 主链一致性范围

- 继续遵循“目标 -> 蓝图 -> 场景 -> 草稿 -> Gate -> ChangeSet -> Publish”；
- 不改变 Canon 写入边界；
- 不定义绕过 Gate / ChangeSet 的捷径。

## 5. MVP 不包含内容

- 不包含自动重算全链路实现；
- 不包含自动失效传播引擎实现；
- 不包含跨章批量重跑调度实现；
- 不包含数据库结构调整；
- 不包含执行级测试用例与发布计划。

## 6. MVP 验收口径（本轮）

本轮 docs-only 验收采用以下标准：
- MVP 范围已文档化且边界清楚；
- 与方向确认、阶段边界文档一致；
- 不越界为实现任务单。

## 7. 结论

下游依赖失效与重跑治理的 MVP 当前仅处于“立项定义完成”状态，
**未进入“可直接开发自动重算能力”状态。**
