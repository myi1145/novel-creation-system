# 03_发布章节成品阅读与导出MVP范围

## 1. 文档目的

定义下一阶段可执行的最小可交付范围，作为后续实现验收口径基础。

## 2. MVP 结论先行

MVP 聚焦单章已发布成品，不扩展到整本书与复杂出版能力。

## 3. MVP 必须覆盖

- 已发布章节阅读页；
- 章节标题 / 章号；
- 正文阅读视图；
- 发布信息；
- 一键复制正文；
- 单章 Markdown 导出；
- 单章 txt 导出。

## 4. MVP 不包含

- 整本书导出；
- PDF / Word 导出；
- 复杂排版模板；
- 批量导出；
- 云分享链接。

## 5. 最小后端要求

- 提供单章已发布成品读取所需最小字段；
- 提供单章导出所需的最小格式化输出能力（Markdown/txt）；
- 保持只读语义，不改变 Publish / ChangeSet 主链协议。

## 6. 最小前端要求

- 提供单章成品阅读页最小布局与信息区块；
- 提供一键复制正文交互反馈；
- 提供 Markdown / txt 导出入口与失败提示。

## 7. 最小测试要求

- 覆盖已发布章节读取与字段完整性；
- 覆盖 Markdown / txt 导出结果基本正确性；
- 覆盖前端复制与导出入口的最小交互回归。

## 8. 验收口径

达到以下条件即视为 MVP 可验收：
- 作者可进入单章成品阅读页稳定查看正文与发布信息；
- 作者可一键复制正文；
- 作者可导出 Markdown 与 txt；
- 未引入整本书导出、复杂排版或云分享相关能力。

## 9. 一句话口径

**这一阶段的 MVP，不是做完整出版系统，而是先让单章成品可阅读、可复制、可导出。**
