# 00_发布章节成品阅读与导出候选方向与问题定义

## 1. 文档目的

在轻量版本差异对比与重发决策阶段完成并签收后，明确下一阶段候选方向，统一问题定义，避免实现前目标漂移。

## 2. 当前已完成能力回顾

当前系统已具备以下基线能力：
- 章节主链（目标 -> 蓝图 -> 场景 -> 草稿 -> Gate -> ChangeSet -> Publish）可执行；
- 发布历史可追踪，发布前一致性验收可执行；
- 轻量版本差异摘要与重发建议已完成最小闭环；
- VersionDiffPage 与相关入口已接线。

上述能力已满足“发布决策前后可核查”的阶段目标，但尚未解决“发布成品可消费”的作者端体验问题。

## 3. 当前发布后作者使用断点

当前发布后仍存在明显断点：
- 已发布章节缺少“成品阅读视图”，作者阅读体验偏管理页；
- 章节正文复制链路不明确，作者拿走成品内容成本高；
- 已发布单章缺少直接导出入口，外部流转依赖手工整理；
- 发布后消费路径与创作路径耦合过紧，不利于交付环节闭环。

## 4. 候选方向对比

### 4.1 发布章节成品阅读页
- 价值：直接解决“发布后如何像作品一样阅读”问题；
- 成本：中等，主要在展示层与最小聚合字段；
- 结论：应优先。

### 4.2 一键复制正文
- 价值：直接降低作者二次使用成本；
- 成本：低，交互与边界清晰；
- 结论：建议纳入本阶段。

### 4.3 Markdown 导出
- 价值：兼容常见编辑器与内容平台；
- 成本：中低，格式约定清晰；
- 结论：建议纳入单章范围。

### 4.4 txt 导出
- 价值：实现简单、通用性高；
- 成本：低；
- 结论：建议纳入单章范围。

### 4.5 单章导出
- 价值：与章节工作流一致，交付闭环最短；
- 成本：可控；
- 结论：本阶段优先。

### 4.6 多章 / 整本书导出
- 价值：高，但依赖目录编排、版本选择、批量容错；
- 成本：高，超出本阶段最小目标；
- 结论：暂不优先。

### 4.7 复杂排版导出
- 价值：偏出版后期；
- 成本：高，牵涉模板体系与格式渲染；
- 结论：暂不优先。

## 5. 本阶段建议优先方向

建议优先方向为：**发布章节成品阅读 + 一键复制正文 + 单章 Markdown/txt 导出**。

该方向能以最小代价补齐“发布后可消费”缺口，并与现有发布主链保持解耦。

## 6. 本阶段暂不优先方向

本阶段暂不优先以下方向：
- 整本书导出与批量导出；
- 复杂排版模板与多格式富导出；
- 云端分享与外链分发。

## 7. 最终问题定义

本阶段需要解决的核心问题是：

**在不改动 Publish 主链语义的前提下，如何让作者对已发布章节实现“可阅读、可复制、可导出”的最小闭环。**

重点结论：

**当前最需要解决的不是继续检查章节，而是让作者能阅读、复制并拿走已经发布的成品章节。**
