# 01_发布章节成品阅读与导出方向确认

## 1. 文档目的

在候选方向对比基础上形成单一主方向，避免后续阶段并行扩散。

## 2. 结论先行

下一阶段方向确认为：**发布章节成品阅读与单章导出**，并以**一键复制正文**作为次方向同步纳入。

## 3. 主方向

**发布章节成品阅读与单章导出**。

主方向要求：
- 面向已发布章节提供成品阅读体验；
- 支持单章导出，优先 Markdown 与 txt；
- 不改变发布主链语义，不引入新发布状态。

## 4. 次方向

**一键复制正文**。

次方向定位：
- 作为成品消费的高频能力，属于 MVP 内必需配套；
- 以最小交互实现，不引入复杂权限与模板扩展。

## 5. 暂不采用方向

本阶段明确暂不采用：
- 整本书导出；
- 复杂排版导出；
- 多格式富导出；
- 云端分享链接。

## 6. 为什么现在做这个方向

原因如下：
- 上一阶段已解决“是否重发”的决策辅助，本阶段应转向“发布成品如何被消费”；
- 作者在发布后仍存在阅读、复制、导出断点，影响交付效率；
- 单章级能力可与章节主链天然对齐，收口风险最低。

## 7. 与 PublishedPage / PublishHistory / VersionDiff 的关系

关系定位如下：
- PublishedPage：保留管理与发布入口定位，不替代成品阅读页；
- PublishHistory：继续承担历史追踪定位，不扩展为成品展示；
- VersionDiff：作为重发辅助能力保留，不再扩展为本阶段主功能。

## 8. 一句话拍板

**下一阶段先让作者能像读正式作品一样查看已发布章节，并能把单章成品复制或导出。**
