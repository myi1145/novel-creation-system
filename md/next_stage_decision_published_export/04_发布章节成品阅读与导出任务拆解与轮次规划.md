# 04_发布章节成品阅读与导出任务拆解与轮次规划

## 1. 文档目的

为后续可能进入实现阶段提供轮次化推进框架，确保先审计、后实现、可收口。

## 2. 轮次规划结论先行

建议按五轮推进：
1. 第 0 轮：审计已发布章节数据来源与 PublishedPage 现状；
2. 第 1 轮：补齐单章成品读取 / 导出接口设计；
3. 第 2 轮：新增成品阅读页与复制能力；
4. 第 3 轮：补齐 Markdown / txt 单章导出；
5. 第 4 轮：测试、文档与阶段验收收口。

## 3. 每轮目标

### 3.1 第 0 轮目标
- 盘点已发布章节可用字段与来源链路；
- 明确 PublishedPage 与成品阅读页职责边界；
- 形成最小数据契约草案。

### 3.2 第 1 轮目标
- 定义单章成品读取协议；
- 定义 Markdown / txt 导出最小接口契约；
- 明确错误码与异常返回最小规范。

### 3.3 第 2 轮目标
- 建立成品阅读页最小信息结构；
- 接入一键复制正文；
- 保持与现有 Published 管理页解耦。

### 3.4 第 3 轮目标
- 完成单章 Markdown 导出；
- 完成单章 txt 导出；
- 补齐导出入口与最小回归验证。

### 3.5 第 4 轮目标
- 汇总接口、页面、交互测试；
- 同步 README / handoff 与阶段文档；
- 输出阶段验收结论。

## 4. 每轮不做事项

- 第 0 轮：不实现接口与页面；
- 第 1 轮：不开发整本书导出；
- 第 2 轮：不扩展复杂排版模板；
- 第 3 轮：不引入 PDF / Word 导出；
- 第 4 轮：不新增跨阶段需求。

## 5. 每轮完成标志

- 第 0 轮：审计结论可支撑接口设计；
- 第 1 轮：接口契约可直接进入实现；
- 第 2 轮：成品阅读与复制链路可演示；
- 第 3 轮：Markdown / txt 导出可用且可回归；
- 第 4 轮：文档、测试、验收口径一致。

## 6. 轮次依赖关系

- 第 0 轮是后续轮次输入前提；
- 第 1 轮完成后第 2 轮与第 3 轮才能稳定推进；
- 第 2/3 轮完成后第 4 轮用于统一收口；
- 不允许跳过第 0 轮直接进入实现。

## 7. 不应出现的执行偏差

- 在第 0 轮审计前直接启动页面开发；
- 在第 1 轮提前承诺整本书导出；
- 在第 2/3 轮扩展为出版级格式系统；
- 在第 4 轮继续扩范围而不签收。

## 8. 与未来 05 执行任务单的关系

本文件是轮次规划，不是执行工单。  
当前阶段不新增 05；仅在 00~04 全部评审通过且确认进入实现阶段后，才可讨论是否生成 05。

## 9. 最终确认结论

当前仅完成“发布章节成品阅读与导出”立项与轮次规划，不代表已进入实现开发。
