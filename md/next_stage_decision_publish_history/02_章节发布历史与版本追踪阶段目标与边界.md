# 02_章节发布历史与版本追踪阶段目标与边界

## 1. 文档目的

明确“章节发布历史与版本追踪”阶段的唯一目标、问题集合与严格边界，防止范围膨胀。

## 2. 阶段定位

本阶段定位为：**立项与边界定义阶段（docs-only）**，用于确定后续实现准入前提，不直接进入功能开发。

## 3. 本阶段唯一目标

**补齐章节发布后的版本来源追踪与发布历史查看能力。**

## 4. 本阶段要解决的问题

本阶段聚焦以下问题：
1. 发布记录不可见；
2. 发布来源不清楚；
3. 发布版本与 draft / changeset 关系不清楚；
4. 当前工作态是否晚于发布态不清楚。

## 5. 本阶段严格边界

本阶段明确不做：
- 不做版本回滚；
- 不做复杂文本 diff；
- 不做多章节发布总览；
- 不改 Publish 核心语义；
- 不自动发布；
- 不自动回滚。

## 6. 允许的最小改动类型

本阶段仅允许：
- 文档级方向确认；
- 文档级边界收口；
- 文档级 MVP 范围与轮次规划定义；
- 仓库入口口径同步（handoff / README / AGENTS）。

## 7. 后置事项

以下事项后置到“实现准入通过后”：
- API 与数据结构补齐；
- 页面与入口实现；
- 测试与验收用例落地；
- 运行期观测与治理指标补齐。

## 8. 最终确认结论

本阶段只完成“发布后可追踪能力”的立项与边界定义，不进入任何后端、前端、数据库、测试逻辑开发。
