# 04_蓝图级人工修订任务拆解与轮次规划

## 1. 文档目的

本文件用于在立项层给出后续若进入实现阶段时的轮次推进框架，避免后续任务直接发散。

本文件是规划件，不是执行工单。

## 2. 轮次规划结论先行

蓝图级人工修订后续建议按四轮推进：

1. **第 0 轮：现状审计与挂接点确认**
2. **第 1 轮：后端最小语义设计与接口草案评审**
3. **第 2 轮：前端最小交互原型与主链回接方案评审**
4. **第 3 轮：实现决策前验收与执行任务单准入评审**

说明：本文件仅定义轮次框架，不代表立即进入第 1~3 轮实现。

## 3. 第 0 轮：现状审计与挂接点确认

目标：
- 复核 `ChapterBlueprint` 在现有流程中的状态语义。
- 确认可复用链路与潜在冲突点。
- 识别最小改动挂接位与高风险区域。

产出：
- 审计结论。
- 候选挂接方案对比。
- 风险清单与边界确认。

## 4. 第 1 轮：后端最小语义设计与接口草案评审

目标：
- 仅在设计层形成蓝图级人工修订接口草案与状态语义草案。
- 明确与 Gate / ChangeSet / Publish 的关系边界。

产出：
- 接口草案文档。
- 审计字段草案。
- 不破坏主链语义的约束清单。

边界：
- 不落业务代码。
- 不改数据库结构。

## 5. 第 2 轮：前端最小交互原型与主链回接方案评审

目标：
- 在设计层确认最小交互路径与回主链路径。
- 明确入口位置、流程状态提示与人工操作闭环。

产出：
- 交互草图或页面流程说明。
- 回接路径说明。
- 与后端语义对齐记录。

边界：
- 不开发正式编辑器。
- 不做体验增强。

## 6. 第 3 轮：实现决策前验收与执行任务单准入评审

目标：
- 基于前述产出判断是否具备进入实现阶段的前置条件。
- 形成“是否可以编写执行任务单”的准入结论。

产出：
- 准入评审结论。
- 若通过，再进入后续执行任务单编写。

边界：
- 本轮仍不产出执行代码。
- 本轮之前不生成实现工单。

## 7. 本文件执行约束

- 轮次顺序不可跳步。
- 任一轮出现范围膨胀信号，应回到上一轮收口。
- 必须始终遵守 Canon / Story State 中枢与 ChangeSet 写入边界。

## 8. 结论

本轮仅完成“可执行的规划框架”，
**尚未进入蓝图级人工修订实现阶段。**

---

## 9. 实现准备阶段补充（增量同步）

在“方向确认与边界立项”完成后，已进入实现准备并执行最小闭环落地，范围严格限定为：
- `ChapterBlueprint` 人工编辑入口；
- 最小审计记录（edit_reason / edited_at / source_type=human_edited）；
- 人工编辑后回接“场景拆解 -> 草稿 -> Gate -> ChangeSet -> Publish”主链。

该补充用于同步阶段衔接，不改变本文件既有边界定义与主链原则。
