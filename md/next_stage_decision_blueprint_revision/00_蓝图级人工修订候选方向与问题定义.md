# 00_蓝图级人工修订候选方向与问题定义

## 1. 文档目的

本文件用于在“文本级人工修订闭环阶段已签收”的前提下，明确下一阶段候选方向、问题定义与筛选依据，为方向确认文档提供上游材料。

本文件是研究与问题定义件，不是执行任务单，不进入代码实现层。

## 2. 当前阶段切换前提

已签收前提如下：
- `ChapterDraft` 文本级人工修订闭环已完成并签收。
- 最小审计与回主链能力已具备。
- 当前主矛盾从“草稿文本接管”转移到“上游规划层可控修订”。

因此，下一阶段应优先选择能增强上游控制力、且不破坏现有主链语义的方向。

## 3. 候选方向

### 3.1 候选方向 A：蓝图级人工修订

定义：围绕 `ChapterBlueprint` 建立人工修订入口与主链回接约束，使作者可在“场景拆解与正文生成之前”校正章节结构与推进策略。

潜在收益：
- 问题前移，降低后续正文返工成本。
- 维持“先蓝图、后正文”的主链约束。
- 能与既有 Gate / ChangeSet / Publish 语义形成分层衔接。

### 3.2 候选方向 B：场景级人工修订

定义：围绕 `SceneCard` 建立人工修订能力，强化场景粒度的人工控制。

潜在收益：
- 可对镜头级叙事进行更细颗粒控制。
- 能进一步提升正文执行阶段的可预期性。

主要风险：
- 颗粒度更细，实施复杂度与协同成本显著更高。
- 在蓝图层尚未立项收口时，容易导致范围膨胀。

### 3.3 候选方向 C：结构化设定卡扩展

定义：优先扩展更多结构化对象类型（如设定卡层）。

潜在收益：
- 增强长期世界观建模能力。
- 提升多题材扩展深度。

主要风险：
- 对当前“修订链路上移”的直接帮助有限。
- 容易偏离当前阶段“先补上游人工控制层”的目标。

## 4. 问题定义（下一阶段要解决什么）

### 4.1 核心问题

在文本级人工修订已签收后，当前核心缺口是：

**作者缺少对章节蓝图这一上游关键对象的受控人工修订能力，导致高价值决策点仍过度后置到草稿阶段。**

### 4.2 问题影响

- 规划偏差常在草稿阶段才被发现，返工成本偏高。
- “目标 -> 蓝图 -> 场景 -> 草稿”链路中，蓝图节点的人机协同杠杆不足。
- 既有主链虽可跑通，但上游可控性仍不充分。

### 4.3 成功标准（立项层）

本阶段成功不以代码实现衡量，而以以下立项标准衡量：
- 方向是否收敛到单一优先方向。
- 阶段目标与边界是否可执行且不越界。
- MVP 范围是否最小、清晰、可验收。
- 轮次规划是否可作为后续执行上游输入。

## 5. 方向筛选标准

为避免重复“多线并行导致失焦”的问题，候选方向筛选采用以下标准：
- 与现有主链语义一致性。
- 对当前核心痛点的直接命中度。
- 增量实现复杂度是否可控。
- 是否便于在不改 Canon 写入原则前提下推进。
- 是否有明确“不做清单”以防范围膨胀。

## 6. 研究结论（供 01 文档确认）

基于上述比较，本轮建议：

**下一阶段优先方向应为“蓝图级人工修订立项”，场景级修订与结构化设定卡扩展继续后置。**

该结论仅用于方向确认输入，不代表已进入实现阶段。
