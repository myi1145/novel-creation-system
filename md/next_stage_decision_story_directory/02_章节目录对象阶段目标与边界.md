# 02_章节目录对象阶段目标与边界

## 1. 文档目的

定义“章节目录对象”阶段的唯一目标、问题范围与严格边界，确保团队按 docs-only 立项推进，不提前进入实现开发。

## 2. 阶段定位

本阶段定位为：**章节目录对象方向确认与边界立项阶段（docs-only）。**

阶段职责是固化 StoryDirectory / ChapterDirectory 的最小设计边界与依赖关系。

## 3. 本阶段唯一目标

**建立 StoryDirectory / ChapterDirectory 的最小设计边界。**

## 4. 本阶段要解决的问题

1. 全书规划尚未落到章节序列；
2. 章节目标缺少目录约束；
3. 章节蓝图缺少上游章节职责；
4. 卡槽候选缺少章节覆盖范围。

## 5. 本阶段严格边界

- 不实现目录生成；
- 不实现目录页面；
- 不接章节蓝图；
- 不生成卡槽候选；
- 不写 Canon；
- 不生成 ChangeSet；
- 不重写章节主链。

## 6. 允许的最小能力类型

在 docs-only 前提下，本阶段仅允许输出：

1. StoryDirectory / ChapterDirectory 对象的阶段定位与边界说明；
2. 目录对象与 StoryPlanning、卡槽候选、ChapterGoal、ChapterBlueprint 的输入关系说明；
3. 目录对象与 Canon / Story State / ChangeSet 的边界约束；
4. 后续实现轮次建议与验收口径。

## 7. 后置事项

以下事项后置，不纳入本阶段：

1. 目录对象 schema 实现与落库；
2. 目录生成与编辑交互实现；
3. 目录摘要接入 ChapterGoal / ChapterBlueprint 实现；
4. 基于目录的卡槽候选生成实现；
5. 目录相关 API、测试与运维收口。

## 8. 最终确认结论

本阶段仅完成“章节目录对象方向与边界”立项收口，输出止于 00~04 决策文档链，不进入功能实现与 05 执行任务单。
