# 04_章节目录对象任务拆解与轮次规划

## 1. 文档目的

给出“章节目录对象”后续实现阶段的轮次化推进框架，确保按 V1 约束增量推进、按轮验收、避免偏离主线。

## 2. 轮次规划结论先行

建议按五轮推进：

1. 第 0 轮：审计 StoryPlanning、ChapterGoal、ChapterBlueprint 与现有章节链路；
2. 第 1 轮：定义 StoryDirectory / ChapterDirectory schema；
3. 第 2 轮：补齐目录对象读取 / 保存的最小闭环；
4. 第 3 轮：补齐目录生成 / 编辑与作者确认；
5. 第 4 轮：将目录摘要接入 ChapterGoal / ChapterBlueprint 输入。

## 3. 推荐五轮推进

### 3.1 第 0 轮：审计 StoryPlanning、ChapterGoal、ChapterBlueprint 与现有章节链路

- 目标：固化目录层的挂接点、输入契约与不可破坏约束；
- 不做：不改数据库、不改 API、不改章节主链业务逻辑；
- 完成标志：形成目录层挂接审计清单与风险清单。

### 3.2 第 1 轮：定义 StoryDirectory / ChapterDirectory schema

- 目标：明确对象字段、状态分层、引用关系与命名映射；
- 不做：不开发目录生成器、不接入页面；
- 完成标志：schema 评审通过并形成统一字段映射表。

### 3.3 第 2 轮：补齐目录对象读取 / 保存的最小闭环

- 目标：打通目录对象最小创建、编辑、读取、保存链路；
- 不做：不接入 ChapterGoal / ChapterBlueprint，不生成卡槽候选；
- 完成标志：目录对象可稳定持久化并可被上游/下游读取。

### 3.4 第 3 轮：补齐目录生成 / 编辑与作者确认

- 目标：形成目录草案、作者编辑与确认的最小闭环；
- 不做：不自动写 Canon，不自动生成 ChangeSet；
- 完成标志：目录对象状态可追踪、可审计、可回看。

### 3.5 第 4 轮：将目录摘要接入 ChapterGoal / ChapterBlueprint 输入

- 目标：让章节目标与蓝图生成显式引用目录摘要；
- 不做：不重写章节主链，不改变 Gate/Publish 语义；
- 完成标志：ChapterGoal / ChapterBlueprint 输入契约已包含目录引用。

## 4. 每轮目标

每轮均需满足以下共性目标：

1. 不破坏 Canon / Story State / ChangeSet 中枢语义；
2. 不破坏既有章节主链稳定性；
3. 每轮产物可被下一轮直接复用；
4. 每轮结束均可独立验收并记录风险。

## 5. 每轮不做事项

跨轮统一不做事项：

1. 不做自动 Canon 写入；
2. 不做自动 ChangeSet 生成与审批；
3. 不做卡槽新卡型扩展；
4. 不做复杂知识图谱；
5. 不做章节主链重构。

## 6. 每轮完成标志

1. 第 0 轮：审计报告签收；
2. 第 1 轮：目录对象 schema 评审通过；
3. 第 2 轮：目录对象最小读写闭环联调通过；
4. 第 3 轮：目录生成与作者确认流程回归通过；
5. 第 4 轮：目录摘要挂接 ChapterGoal / ChapterBlueprint 验收通过。

## 7. 轮次依赖关系

1. 第 0 轮是第 1~4 轮前提；
2. 第 1 轮完成后，方可进入第 2/3 轮；
3. 第 2 与第 3 轮完成后，方可进入第 4 轮挂接；
4. 不允许跳过前置轮次直接并行开发。

## 8. 不应出现的执行偏差

1. 未完成目录对象边界审计即启动实现；
2. 将目录对象当作 Canon 直接写入；
3. 在目录层未落稳前直接执行卡槽候选生成；
4. 在 00~04 未评审前提前编写执行任务单；
5. 以“效率”为由重写章节主链。

## 9. 与未来 05 执行任务单的关系

本文件用于阶段规划，不是执行工单。仅当 story_directory 00~04 决策链评审通过并明确进入实现阶段后，才可编制 `05_Codex执行任务单_章节目录对象.md`。

## 10. 最终确认结论

当前阶段应先完成“章节目录对象”立项收口，并以 00~04 作为后续实现的唯一上游输入。评审通过前，不进入实现开发。
