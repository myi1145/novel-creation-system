# 01_轻量版本差异对比与重发决策方向确认

## 1. 文档目的

在候选方向基础上完成拍板，明确主方向、次方向与暂不采用方向，作为阶段目标与 MVP 范围定义的上游依据。

## 2. 结论先行

下一阶段主方向确认为：**轻量版本差异对比**。  
下一阶段次方向确认为：**重发决策辅助提示**。

## 3. 主方向

### 3.1 轻量版本差异对比

主方向定义：
- 面向“当前工作态 vs 最近发布版”提供轻量差异摘要；
- 差异表达以结构化分类为主，不进入复杂全文比对；
- 输出可被作者快速消费的改动影响提示。

主方向目标不是“完整版本 diff 平台”，而是先建立可解释的差异认知。

## 4. 次方向

### 4.1 重发决策辅助提示

次方向定义：
- 在差异摘要基础上提供“建议重发/可暂缓/需人工判断”的辅助提示；
- 提供重发前重点检查项，降低遗漏关键风险概率。

次方向不包含自动重发执行，不替代作者最终判断。

## 5. 暂不采用方向

本阶段暂不采用：
- 复杂文本级 diff 引擎；
- 版本回滚；
- 自动重发；
- Publish 逻辑改造；
- ChangeSet 逻辑改造。

## 6. 为什么现在做这个方向

原因如下：
1. 发布历史与来源追踪阶段已完成并签收，基础可见性问题已解决；
2. 当前核心短板已转移为“差异可理解与重发可决策”；
3. 若缺少轻量差异与决策辅助，作者需要高成本人工对照，协作效率与发布质量判断不稳定；
4. 该方向可复用已建发布历史基线，以低风险增量推进。

## 7. 与既有 Publish / ChangeSet / Gate 的关系

- 与 Publish 的关系：只提供重发建议，不改变发布执行语义；
- 与 ChangeSet 的关系：只消费既有变更信息，不改变审批/应用边界；
- 与 Gate 的关系：补充重发前关注点提示，不替代既有质量闸门。

## 8. 一句话拍板
