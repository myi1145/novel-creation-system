# 02_轻量版本差异对比与重发决策阶段目标与边界

## 1. 文档目的

明确“轻量版本差异对比与重发决策”阶段的唯一目标、问题集合与严格边界，防止范围膨胀。

## 2. 阶段定位

本阶段定位为：**立项与边界定义阶段（docs-only）**，用于确定后续实现准入前提，不直接进入功能开发。

## 3. 本阶段唯一目标

**定义“轻量差异摘要 + 重发辅助决策”的最小立项口径。**

## 4. 本阶段要解决的问题

本阶段聚焦以下问题：
1. 当前工作态相对最近发布版“改了什么”不可快速理解；
2. 改动影响等级缺少统一判断口径；
3. 是否建议重发缺少稳定辅助；
4. 重发前重点核查项缺少统一提示。

## 5. 本阶段严格边界

本阶段明确不做：
- 不改后端代码；
- 不改前端代码；
- 不实现 diff 功能；
- 不实现版本回滚；
- 不实现自动重发；
- 不改 Publish 逻辑；
- 不改 ChangeSet 逻辑；
- 不新增 05 执行任务单。

## 6. 允许的最小改动类型

本阶段仅允许：
- 文档级阶段签收；
- 文档级方向确认；
- 文档级边界收口；
- 文档级 MVP 范围与轮次规划定义；
- 仓库入口口径同步（handoff / README / AGENTS）。

## 7. 后置事项

以下事项后置到“实现准入通过后”：
- 轻量差异计算规则与字段协议实现；
- 前后端展示与交互实现；
- 测试与验收用例落地；
- 观测指标与治理策略补齐。

## 8. 最终确认结论

本阶段只完成“发布历史阶段签收 + 下一阶段立项收口”，不进入任何代码实现工作。
