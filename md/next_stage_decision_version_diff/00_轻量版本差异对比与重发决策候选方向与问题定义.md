# 00_轻量版本差异对比与重发决策候选方向与问题定义

## 1. 文档目的

在“章节发布历史与版本追踪”阶段实现完成并签收后，明确下一阶段候选方向与问题定义，避免直接跳入高风险版本回滚或复杂版本管理。

## 2. 当前已具备能力回顾（签收基线）

当前系统已具备并可复用的发布后基线能力：
- 章节最近一次发布记录可查看；
- 历史发布记录可查看；
- 发布来源草稿可追踪；
- 发布来源 ChangeSet 可追踪；
- 当前工作态是否晚于发布态可提示。

以上能力回答了“发布过什么、来自哪里、现在是否落后”，但尚未回答“到底改了什么、是否值得重发”。

## 3. 当前断点与核心问题

作者当前仍存在稳定决策断点：
1. 无法低成本理解当前工作态相对最近发布版的改动范围；
2. 无法快速区分“文案微调”与“剧情/设定影响改动”；
3. 无法形成一致口径判断“是否建议重新发布”；
4. 重发前缺少明确的重点核查提示。

## 4. 候选方向对比

### 4.1 候选方向 A：轻量版本差异摘要
- 价值：用结构化摘要回答“改了哪些类别、影响面大致如何”。
- 风险：若范围失控，易滑向复杂全文 diff 引擎。

### 4.2 候选方向 B：重发建议分级提示
- 价值：给出“建议重发 / 可暂缓 / 需人工判断”的辅助分级。
- 风险：若做成自动决策，会越界影响 Publish 语义。

### 4.3 候选方向 C：重发前重点核查清单
- 价值：在重发前提示应重点复核的剧情、设定、一致性项。
- 风险：若扩展过度，可能变成新一套质量闸门系统。

### 4.4 候选方向 D：版本回滚与自动重发
- 价值：直接提供执行能力。
- 风险：高风险语义变更，当前阶段不具备准入条件。

## 5. 本阶段建议优先方向

建议优先组合为：
- **轻量版本差异摘要（A）**；
- **重发建议分级提示（B）**；
- **重发前重点核查清单（C）**。

该组合先补齐“差异可理解 + 决策可辅助 + 风险可预警”的最小闭环，不触碰发布执行主链。

## 6. 本阶段暂不优先方向

本阶段暂不优先：
- 复杂文本逐段 diff 视图；
- 版本回滚；
- 自动重发；
- Publish / ChangeSet 核心语义改造。

## 7. 最终问题定义

本阶段最终问题定义为：

**在不修改 Publish 与 ChangeSet 主链语义的前提下，建立“当前工作态相对最近发布版”的轻量差异摘要与重发决策辅助口径。**

一句话结论：

**先让作者看懂“改了什么、影响多大、是否建议重发”，再讨论回滚与自动化执行。**
