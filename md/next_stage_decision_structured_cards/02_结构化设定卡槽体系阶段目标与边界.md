# 02_结构化设定卡槽体系阶段目标与边界

## 1. 文档目的

定义结构化设定卡槽体系阶段的唯一目标、问题范围与严格边界，防止任务在立项阶段失控扩张。

## 2. 阶段定位

本阶段定位为：**结构化设定卡槽体系立项阶段（docs-only）**。

阶段职责是确认方向、边界与 MVP 口径，不进入功能实现。

## 3. 本阶段唯一目标

**建立角色卡与术语卡的最小结构化卡槽体系设计。**

## 4. 本阶段要解决的问题

- 角色设定散落在章节中，难以集中维护；
- 术语设定缺少统一解释，跨章节易出现漂移；
- 作者无法集中查看核心设定资产；
- 后续章节难以稳定复用既有设定。

## 5. 本阶段严格边界

- 不做全量卡槽；
- 不做复杂知识图谱；
- 不做自动抽取；
- 不做卡片自动入史；
- 不绕过 ChangeSet；
- 不直接改 Canon 主逻辑。

补充边界：本阶段不改后端代码、不改前端代码、不改数据库、不新增接口与页面。

## 6. 允许的最小能力类型

在 docs-only 前提下，允许输出的仅为：

1. 角色卡与术语卡的最小字段建议；
2. 卡槽与 Canon / Story State / ChangeSet 的职责边界；
3. 最小读写行为约束（人工创建/编辑、审计来源、入史门禁）；
4. 后续实现轮次建议与验收口径。

## 7. 后置事项

以下事项后置到后续阶段，不纳入本阶段：

- 势力卡 / 地点卡 / 物品卡 / 关系卡 / 伏笔卡；
- 图谱可视化与复杂关系推理；
- 自动全文抽取、自动批量同步；
- 卡槽驱动的高级推荐、自动改写、自动回填。

## 8. 最终确认结论

当前阶段仅完成“结构化设定卡槽体系”立项收口，严格限定为角色卡与术语卡的最小设计边界，不提前进入开发实现。
