# 03_结构化设定卡槽体系MVP范围

## 1. 文档目的

定义结构化设定卡槽体系在后续实现阶段可执行的最小交付范围，确保 MVP 可落地、可验收、可审计。

## 2. MVP 结论先行

MVP 只覆盖角色卡与术语卡的最小结构化能力，不扩展为全量设定库或图谱系统。

## 3. MVP 必须覆盖

- 角色卡最小字段；
- 术语卡最小字段；
- 卡片列表；
- 卡片详情；
- 人工创建 / 编辑的最小设想；
- 与 ChangeSet / Canon 的边界关系；
- 后续可被章节上下文引用。

## 4. MVP 不包含

- 势力卡；
- 地点卡；
- 物品卡；
- 伏笔卡；
- 关系图谱；
- 自动抽取；
- 自动同步全文。

## 5. 角色卡建议字段

角色卡（CharacterCard）最小字段建议：

- 名称
- 别名
- 身份定位
- 人物简介
- 性格关键词
- 关系备注
- 当前状态
- 首次出现章节
- 最近更新来源
- 是否进入 Canon

字段约束建议：
- “最近更新来源”必须可追溯到章节、人工修订或 ChangeSet；
- “是否进入 Canon”仅反映状态，不代表可绕过 ChangeSet 直接写入。

## 6. 术语卡建议字段

术语卡（TerminologyCard）最小字段建议：

- 术语名称
- 术语类型
- 定义
- 使用规则
- 例句 / 用法
- 首次出现章节
- 最近更新来源
- 是否进入 Canon

字段约束建议：
- “定义 + 使用规则”作为术语一致性的核心校验输入；
- 术语口径变更应保留版本化审计线索。

## 7. 最小后端要求

后续若进入实现，最小后端要求应为：

1. 提供角色卡 / 术语卡最小读模型（列表、详情）；
2. 提供人工创建 / 编辑的最小写模型；
3. 写入 Canon 仍通过 ChangeSet 门禁，不新增绕行路径；
4. 支持来源追踪字段落审计记录；
5. 不改发布链路语义。

## 8. 最小前端要求

后续若进入实现，最小前端要求应为：

1. 卡片列表页（最小筛选与定位）；
2. 卡片详情页（字段分区展示）；
3. 人工创建 / 编辑表单（最小必填校验）；
4. Canon 入史状态可见、来源可见、边界提示可见；
5. 不扩展图谱大屏与复杂批量工具。

## 9. 最小测试要求

后续若进入实现，最小测试要求应为：

1. 角色卡 / 术语卡字段完整性与必填约束测试；
2. 列表 / 详情读取一致性测试；
3. 人工创建 / 编辑的最小流程测试；
4. ChangeSet 入史边界测试（禁止绕过）；
5. 卡片被章节上下文引用的最小联通测试。

## 10. 验收口径

达到以下条件可判定 MVP 验收通过：

- 角色与术语设定可结构化存放与查看；
- 卡片信息具备来源与状态追踪；
- Canon 入史边界清晰且未绕过 ChangeSet；
- 章节上下文可最小引用卡片信息；
- 未引入全量卡槽、图谱系统与自动抽取。

**这一阶段的 MVP，不是做完整知识库，而是先让角色与术语设定有地方放、有结构、有来源、有边界。**
