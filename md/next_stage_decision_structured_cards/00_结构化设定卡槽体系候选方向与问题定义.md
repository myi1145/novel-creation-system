# 00_结构化设定卡槽体系候选方向与问题定义

## 1. 文档目的

在作者端主路径体验完成一轮产品化收口后，明确下一阶段是否进入“结构化设定卡槽体系”立项，并给出候选方向对比、优先顺序与问题定义，为后续 01~04 文档提供统一输入。

## 2. 当前已完成能力回顾

基于 current handoff、近期阶段文档与 V1 体系审计，当前系统已完成并签收：

- 章节主链（目标 -> 蓝图 -> 场景 -> 草稿 -> Gate -> ChangeSet -> Publish）闭环；
- 草稿 / 蓝图 / 场景人工修订与回链；
- 下游依赖失效识别与重跑治理；
- 发布前一致性检查、发布历史与版本差异；
- 已发布章节阅读与导出最小能力；
- 作者端术语降噪与主路径体验收口。

结论：当前已经具备进入“下一条主线立项”的工程基础，不应继续把主精力投入发布后能力的无限细化。

## 3. 当前系统进入卡槽体系前的断点

当前断点不在“能否生成章节”，而在“核心设定资产能否稳定沉淀”：

1. 角色事实大量散落在章节文本、摘要与临时上下文中；
2. 术语定义缺少统一口径，跨章节复用时易漂移；
3. 作者缺少集中查看与修订核心设定的结构化入口；
4. 章节推进对长期一致性的依赖已提升，但设定资产治理层仍薄弱；
5. 若直接扩展更多发布后能力，会继续放大“文本有、资产无”的结构性风险。

## 4. 候选方向对比

### 4.1 角色卡
- 价值：最高。直接承接长篇一致性中最核心的人物信息治理。
- 复杂度：中。字段边界可先从最小集合起步。
- 与 Canon 关系：强。角色关键事实必须可追溯、可审计。
- 是否建议首批：是。

### 4.2 术语卡
- 价值：高。可显著降低世界观术语漂移与误用。
- 复杂度：低到中。可先做定义与使用规则的最小结构。
- 与 Canon 关系：强。术语口径应纳入正史约束。
- 是否建议首批：是。

### 4.3 势力卡
- 价值：中高。适合中后期扩展。
- 复杂度：中高。涉及组织层级、关系网络与时间演化。
- 是否建议首批：否，后置。

### 4.4 地点卡
- 价值：中。对部分题材重要，但首批收益不及角色/术语。
- 复杂度：中。常与时间线、路线、事件绑定。
- 是否建议首批：否，后置。

### 4.5 物品卡
- 价值：中。偏题材相关，通用性较角色/术语弱。
- 复杂度：中。需处理状态变化与持有关系。
- 是否建议首批：否，后置。

### 4.6 关系卡
- 价值：中高。对群像创作有价值。
- 复杂度：高。容易演化成图谱项目并引发范围膨胀。
- 是否建议首批：否，后置。

### 4.7 伏笔卡
- 价值：中高。对悬念管理关键。
- 复杂度：高。强依赖时间线、章节回收与验证机制。
- 是否建议首批：否，后置。

### 4.8 全量设定库
- 价值：理论高，但当前阶段风险高。
- 复杂度：极高。会同时触发对象、流程、接口、UI、测试全面扩张。
- 是否建议首批：否，不作为当前方向。

## 5. 本阶段建议优先方向

本阶段建议优先：

1. 角色卡（CharacterCard）
2. 术语卡（TerminologyCard）

理由：二者在跨章节一致性中收益最高、字段可先最小化、且最容易与现有 Canon / Story State / ChangeSet 机制对齐，不破坏主链。

## 6. 本阶段暂不优先方向

本阶段暂不优先：势力卡、地点卡、物品卡、关系卡、伏笔卡、一次性全量设定库。

后置原则：先把最小卡槽底座做对，再扩类型，不在立项阶段提前承诺全量对象。

## 7. 最终问题定义

当前最需要解决的是：**长篇创作中的角色、术语等核心设定资产需要结构化沉淀、可追踪、可复用，而不是继续停留在章节文本与临时上下文中。**

因此，下一阶段应先完成“结构化设定卡槽体系”立项收口，先聚焦角色卡与术语卡的最小可行方向，为后续实现阶段提供边界明确、可审计、可迭代的输入。
