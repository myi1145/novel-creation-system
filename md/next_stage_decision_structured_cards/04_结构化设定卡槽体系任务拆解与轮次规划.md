# 04_结构化设定卡槽体系任务拆解与轮次规划

## 1. 文档目的

给出结构化设定卡槽体系后续可能进入实现阶段时的轮次化推进框架，确保“先审计、后实现、可验收、可收口”。

## 2. 轮次规划结论先行

推荐按五轮推进：

1. 第 0 轮：审计现有结构化对象与 Canon 记录方式；
2. 第 1 轮：确定角色卡 / 术语卡 schema 与边界；
3. 第 2 轮：补齐只读卡片列表与详情；
4. 第 3 轮：补齐人工创建 / 编辑与审计；
5. 第 4 轮：接入章节上下文引用与最小验收。

## 3. 每轮目标

### 3.1 第 0 轮目标
- 盘点现有 Schema、状态字段、ChangeSet 记录链路；
- 明确角色卡/术语卡与既有对象映射关系；
- 输出不可破坏约束清单。

### 3.2 第 1 轮目标
- 固化角色卡/术语卡最小字段 schema；
- 明确“是否进入 Canon”状态定义与变更门禁；
- 固化读写接口边界草案。

### 3.3 第 2 轮目标
- 提供卡片列表与详情只读能力；
- 校验字段展示一致性与来源追踪可见性；
- 保持只读阶段不触碰入史逻辑。

### 3.4 第 3 轮目标
- 补齐人工创建 / 编辑能力；
- 补齐最小审计记录（来源、操作者、时间、变更摘要）；
- 与 ChangeSet 入史流程对齐。

### 3.5 第 4 轮目标
- 接入章节上下文最小引用能力；
- 完成边界回归与验收收口；
- 输出阶段签收结论与后置清单。

## 4. 每轮不做事项

- 第 0 轮：不做接口开发、不做页面开发；
- 第 1 轮：不扩展势力/地点/物品/伏笔卡；
- 第 2 轮：不做图谱大屏、不做自动抽取；
- 第 3 轮：不做批量自动入史、不做自动改写主线；
- 第 4 轮：不新增跨阶段需求、不启动全量卡槽。

## 5. 每轮完成标志

- 第 0 轮：审计报告可直接支撑 schema 设计；
- 第 1 轮：schema 与边界经评审可进入开发；
- 第 2 轮：列表/详情可稳定读取并展示关键字段；
- 第 3 轮：人工创建/编辑链路可用且有审计线索；
- 第 4 轮：章节引用与最小回归通过并完成阶段签收。

## 6. 轮次依赖关系

- 第 0 轮是第 1~4 轮前提；
- 第 1 轮完成后，第 2/3 轮方可进入实现；
- 第 2 与第 3 轮完成后，第 4 轮进行联通验收；
- 不允许跳过第 0/1 轮直接实现。

## 7. 不应出现的执行偏差

- 未完成审计即开始编码；
- 在 MVP 阶段承诺全量卡槽；
- 通过“临时快捷路径”绕过 ChangeSet 入史；
- 把卡槽项目扩成知识图谱大工程；
- 在未签收前提前生成 05 执行任务单并启动开发。

## 8. 与未来 05 执行任务单的关系

本文件属于轮次规划，不是执行工单。本阶段只产出 00~04。仅当 00~04 评审通过且明确进入实现阶段后，才可讨论并编制 05 执行任务单。

## 9. 最终确认结论

当前阶段已完成结构化设定卡槽体系的立项拆解与轮次规划，下一步应先完成评审收口，再决定是否进入 05 执行任务单编制。
