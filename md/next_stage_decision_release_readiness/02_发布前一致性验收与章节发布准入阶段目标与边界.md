# 02_发布前一致性验收与章节发布准入阶段目标与边界

## 1. 文档目的

本文件定义当前阶段目标、纳入项、排除项与完成判定，作为后续 MVP 与轮次规划的约束基线。

## 2. 阶段定位

当前阶段定位为：

**发布前一致性验收 / 章节发布准入的方向确认与边界立项阶段（docs-only）。**

## 3. 本阶段唯一目标

**完成“章节是否可安全发布”的统一判定框架立项收口。**

展开如下：
- 明确发布前典型不一致状态；
- 明确信息汇总、准入规则、阻断策略的分层边界；
- 明确本阶段 MVP 与后续实现准入前置条件。

## 4. 本阶段纳入范围（立项层）

- 发布前不一致状态分类定义；
- 章节发布准入口径分层定义；
- 与既有 Gate / ChangeSet / Publish 关系边界定义；
- MVP 最小范围定义；
- 后续轮次推进框架定义。

## 5. 本阶段严格边界

### 5.1 不进入实现

- 不实现发布准入面板前后端功能；
- 不实现统一一致性检查聚合接口；
- 不实现自动阻断发布逻辑；
- 不实现自动修复与自动重算策略。

### 5.2 不改核心业务语义

- 不改 `Gate / ChangeSet / Publish` 核心规则；
- 不改 `chapter-cycle / chapter-sequence` 语义；
- 不改 Canon 只能通过 ChangeSet 写入原则。

### 5.3 不做结构层改造

- 不改数据库结构；
- 不改测试逻辑；
- 不改 `.github/workflows`。

### 5.4 不输出执行工单

- 本阶段不新增 `05_Codex执行任务单_发布前一致性验收与章节发布准入.md`；
- 本阶段结论不等价于“立即开发”。

## 6. 本阶段完成判定

本阶段完成与否仅看以下条件是否同时成立：
- 方向已确认且无并行主线冲突；
- 阶段边界已明确，存在稳定不做清单；
- MVP 范围与轮次规划可承接后续实现评审；
- 仓库入口口径（handoff / README / AGENTS）与阶段结论一致。

## 7. 结论

本阶段是“立项收口阶段”，不是“实现启动阶段”。
