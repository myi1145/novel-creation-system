# 03_发布前一致性验收与章节发布准入MVP范围

## 1. 文档目的

本文件在立项层定义“发布前一致性验收 / 章节发布准入”最小可成立范围，不定义代码任务。

## 2. MVP 结论先行

本阶段 MVP 应收敛为：

**先定义章节发布前一致性信息汇总与准入提示分层口径，不提前定义硬阻断与自动修复实现。**

## 3. MVP 必须覆盖的问题

- 如何统一展示章节发布前关键状态；
- 如何表达“可发布 / 风险可发布 / 不满足准入建议先处理”三类状态；
- 如何区分信息汇总、准入规则、自动阻断的边界；
- 如何保证不破坏既有主链与入史规则。

## 4. MVP 最小范围（立项语义）

### 4.1 发布前信息汇总（必须）

至少纳入以下状态项：
- 是否存在 stale 且未人工确认重跑的下游对象；
- 最新 Gate 是否通过（含关键失败等级）；
- ChangeSet 是否已完成 apply；
- 当前工作态与已发布态是否存在显著不一致（仅定义判定语义，不定义实现算法）。

### 4.2 准入提示分层（必须）

- 第一层：信息提示（不影响准入）；
- 第二层：准入警示（建议处理后再发布）；
- 第三层：阻断候选（仅保留为后续实现评审议题，本阶段不落地）。

### 4.3 主链一致性（必须）

- 持续遵循 `目标 -> 蓝图 -> 场景 -> 草稿 -> Gate -> ChangeSet -> Publish`；
- 不提供绕过 Gate / ChangeSet 的发布捷径；
- 不改变 Publish 既有责任边界。

## 5. MVP 不包含内容

- 不包含发布准入面板实现；
- 不包含统一聚合接口实现；
- 不包含自动阻断发布实现；
- 不包含自动修复、自动重算与跨章调度实现；
- 不包含数据库、测试与 CI 工作流改造。

## 6. MVP 验收口径（本轮）

- MVP 范围已文档化且边界清楚；
- 与 00~02、04 文档一致；
- 未越界为执行工单。

## 7. 结论

当前 MVP 仅处于“立项定义完成”状态，
**尚未进入“发布准入功能可直接开发”状态。**
