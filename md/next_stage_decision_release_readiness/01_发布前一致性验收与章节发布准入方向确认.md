# 01_发布前一致性验收与章节发布准入方向确认

## 1. 文档目的

在候选方向基础上，确认本阶段唯一主方向，并说明不采用其他方向的原因。

## 2. 方向确认结论

**主方向：发布前一致性验收 / 章节发布准入立项优先。**

本阶段目标是先形成统一判定口径：
- 章节当前信息是否一致；
- 是否满足最小发布准入条件；
- 哪些问题仅提示、哪些问题影响准入、哪些问题未来可升级为阻断。

## 3. 为什么不是继续扩自动重算

- stale / recompute 治理闭环已完成并签收，继续扩展不再是当前唯一主矛盾；
- 自动重算解决的是“重跑效率”，不是“发布准入判定统一性”；
- 若准入口径未先收敛，自动重算将放大判定分歧。

## 4. 为什么不是直接做全功能发布控制台

- 统一口径尚未固化，直接实现会导致范围不受控；
- 自动阻断、自动修复与平台化控制台属于后置能力；
- 当前阶段应先建立“可审计、可解释、可分层”的决策基线。

## 5. 方向确认后的阶段重点

- 聚焦“信息汇总视图 + 准入提示分层”的立项定义；
- 明确与 Gate / ChangeSet / Publish 的关系边界；
- 输出 MVP 与轮次规划，但不输出执行工单。

## 6. 当前不采用策略

- 不把发布准入面板写成“已确定立即开发”；
- 不并行立“自动发布”“平台化控制台重构”主线；
- 不越级生成 Codex 执行任务单。
