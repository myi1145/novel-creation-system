# 01_全书级规划与卡槽种子生成方向确认

## 1. 文档目的

在候选方向对比基础上完成阶段拍板，统一下一阶段主方向、MVP 首批范围与暂不采用方向，避免团队再次偏回“卡槽维护工具扩展优先”的路径。

## 2. 结论先行

下一阶段主方向确认为：**全书级规划与卡槽种子生成**。

本阶段定位为 docs-only 立项，不进入功能实现。

## 3. 主方向

主方向定义：

**以全书级对象为上游，先形成世界观、主线大纲、分卷/阶段规划，再生成四类基础卡槽候选种子，并通过作者确认后衔接后续 ChangeSet 入史。**

## 4. 第一批 MVP 方向

第一批 MVP 方向限定为：

1. 全书世界观规划；
2. 全书主线大纲；
3. 分卷 / 阶段规划；
4. 核心角色种子；
5. 核心势力种子；
6. 核心地点种子；
7. 核心术语种子。

## 5. 暂不采用方向

当前阶段明确暂不采用：

- 继续扩新卡型；
- 继续扩导入导出；
- 自动全文抽取建卡；
- LLM 自动入 Canon；
- 复杂知识图谱；
- 重写整套章节主链。

## 6. 为什么现在做这个方向

1. 现有章节级与发布后能力已较完整，系统短板转为上游规划层；
2. 仅靠人工维护卡槽无法稳定支撑长篇创作的一致性与节奏控制；
3. 章节蓝图需要来自全书级目标与分阶段推进约束，而非孤立生成；
4. docs-only 先行可锁定边界，避免直接开发带来 Canon 写入风险。

## 7. 与 Canon / Story State / ChangeSet 的关系

- 全书规划对象与卡槽种子默认处于 Working State，不自动进入 Canon；
- Canon 仍保持“仅能通过 ChangeSet 写入”的唯一入口；
- Story State 负责承载规划态、候选态、确认态与发布态的分层状态；
- 本阶段仅定义边界，不引入任何“直接入史快捷路径”。

## 8. 与章节蓝图 / 场景 / 草稿的关系

- 全书规划摘要将作为 ChapterBlueprint 的上游输入依据；
- 章节蓝图据此做章节目标与冲突推进分解；
- 场景与草稿继续沿现有主链执行，不在本阶段重写；
- 通过“先全书规划、后章节蓝图”补齐当前上游缺口。

## 9. 一句话拍板

**下一阶段先让系统在写章节前形成全书世界观、大纲、阶段规划和核心设定种子，再让章节蓝图基于这些资产推进。**

## 10. 阶段依赖补充（2026-04-15）

在不推翻本文件主方向的前提下，补充执行依赖：

1. 卡槽候选生成前，必须先建立 StoryDirectory / ChapterDirectory；
2. StoryDirectory / ChapterDirectory 位于 StoryPlanning 之后、卡槽候选与 ChapterBlueprint 之前；
3. 后续实现入口应先遵循 `md/next_stage_decision_story_directory/00~04`，再回到本文件定义的卡槽候选链路。
