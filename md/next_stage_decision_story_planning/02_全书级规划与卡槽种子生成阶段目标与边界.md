# 02_全书级规划与卡槽种子生成阶段目标与边界

## 1. 文档目的

定义“全书级规划与卡槽种子生成”阶段的唯一目标、问题范围与严格边界，确保后续任务不偏移为主链重写或自动入史开发。

## 2. 阶段定位

本阶段定位为：**全书级规划与卡槽种子生成立项阶段（docs-only）。**

阶段职责是固化方向与边界，不进入后端、前端、数据库实现。

## 3. 本阶段唯一目标

**建立全书级规划对象与卡槽种子生成的最小设计边界。**

## 4. 本阶段要解决的问题

1. 世界观没有全书级对象；
2. 全书大纲 / 分卷规划缺位；
3. 章节蓝图缺少上游依据；
4. 四类基础卡槽仍偏人工维护表；
5. 导入导出能力不应承担内部设定生成主路径。

## 5. 本阶段严格边界

- 不重写章节主链；
- 不自动写 Canon；
- 不绕过 ChangeSet；
- 不做复杂知识图谱；
- 不做自动全文抽取；
- 不做 LLM 直接入史；
- 不继续扩新卡型；
- 不把导入导出扩展为当前主战场。

## 6. 允许的最小能力类型

在 docs-only 前提下，本阶段仅允许输出：

1. 全书级规划对象定义建议（世界观/大纲/分卷）；
2. 卡槽种子对象定义建议（角色/术语/势力/地点）；
3. 卡槽三态定义建议（候选卡、作者确认卡、已入 Canon 正式卡）；
4. 与 ChapterBlueprint 的最小输入关系建议；
5. 与 Canon / Story State / ChangeSet 的边界约束；
6. 后续实现轮次建议与验收口径。

## 7. 后置事项

以下事项后置，不纳入本阶段：

- 规划对象生成能力实现；
- 卡槽种子自动生成实现；
- 作者确认流程 UI/接口实现；
- 自动 ChangeSet 生成与自动审批；
- 复杂图谱与高级推理模块。

## 8. 最终确认结论

当前阶段仅完成“全书级规划与卡槽种子生成”立项收口。阶段输出止于 00~04 决策文档链，不进入 05 执行任务单，不进入实现开发。
