# 03_全书级规划与卡槽种子生成MVP范围

## 1. 文档目的

定义“全书级规划与卡槽种子生成”在后续实现阶段的最小可交付范围，确保 MVP 可落地、可验收、可与现有章节主链平滑衔接。

## 2. MVP 结论先行

MVP 聚焦“章节蓝图之前”的上游对象补齐，不扩展为自动入史系统，不重做整套创作架构。

## 3. MVP 必须覆盖

1. 全书世界观规划对象；
2. 全书主线大纲对象；
3. 分卷 / 阶段规划对象；
4. 核心角色种子；
5. 核心术语种子；
6. 核心势力种子；
7. 核心地点种子；
8. 卡槽候选生成；
9. 作者确认 / 编辑的最小设想；
10. 与 ChapterBlueprint 的最小引用关系。

## 4. MVP 不包含

- 自动写 Canon；
- 自动生成 ChangeSet；
- 自动全文抽取；
- 复杂知识图谱；
- 重写 chapter-cycle；
- 全量卡槽体系重构。

## 5. 最小对象建议

建议在后续实现时优先定义以下对象族：

1. `BookWorldviewPlan`：全书世界观规划对象；
2. `BookOutlinePlan`：全书主线大纲对象；
3. `VolumeStagePlan`：分卷/阶段规划对象；
4. `CardSeedCandidate`：卡槽候选种子对象；
5. `CardSeedReviewed`：作者确认后的种子对象；
6. `CardCanonCandidate`：待 ChangeSet 入史的正式候选对象。

对象状态建议：
- 候选卡（candidate）-> 作者确认卡（reviewed）-> 已进入 Canon 的正式卡（canon）。

## 6. 最小后端要求

后续若进入实现，最小后端要求为：

1. 全书规划对象的创建/更新/读取最小接口；
2. 从全书规划生成四类卡槽候选的最小服务；
3. 候选卡到作者确认卡的状态流转接口；
4. 与 ChapterBlueprint 组装层的最小读取接口；
5. 明确禁止任何绕过 ChangeSet 的 Canon 直写路径。

## 7. 最小前端要求

后续若进入实现，最小前端要求为：

1. 全书规划对象的最小录入与编辑入口；
2. 卡槽候选列表与作者确认入口；
3. 卡槽三态可视标识（候选/确认/入史）；
4. ChapterBlueprint 输入来源提示（来自全书规划摘要与确认卡槽）。

## 8. 最小测试要求

后续若进入实现，最小测试要求为：

1. 全书规划对象 Schema 与状态流转测试；
2. 卡槽候选生成一致性测试；
3. 候选 -> 确认 -> 待入史流程测试；
4. ChapterBlueprint 输入挂接测试；
5. Canon/ChangeSet 边界不被绕过测试。

## 9. 验收口径

满足以下条件可判定 MVP 验收通过：

- 章节生成前可获取稳定的全书规划摘要；
- 四类核心卡槽可由全书规划生成候选并经作者确认；
- 章节蓝图已可引用规划摘要与确认卡槽；
- Canon 仍只通过 ChangeSet 入史，未出现绕行路径。

一句话口径：

**这一阶段的 MVP，不是重做整套创作系统，而是先把全书规划与设定种子放到章节蓝图之前。**
