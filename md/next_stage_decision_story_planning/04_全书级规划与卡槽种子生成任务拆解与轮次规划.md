# 04_全书级规划与卡槽种子生成任务拆解与轮次规划

## 1. 文档目的

给出“全书级规划与卡槽种子生成”在后续实现阶段的轮次化推进框架，确保按 V1 原则增量推进、分轮验收、边界可控。

## 2. 轮次规划结论先行

建议按五轮推进：

1. 第 0 轮：审计项目创建、章节目标、蓝图生成与卡槽现状；
2. 第 1 轮：定义全书规划对象与卡槽种子 schema；
3. 第 2 轮：补齐全书规划生成 / 编辑的最小入口；
4. 第 3 轮：补齐从规划生成卡槽候选与作者确认；
5. 第 4 轮：将规划摘要与确认卡槽接入章节蓝图输入。

## 3. 推荐五轮推进

### 3.1 第 0 轮：审计项目创建、章节目标、蓝图生成与卡槽现状

- 目标：固化当前输入链路、对象边界与可挂接点；
- 不做：不改接口、不改数据库、不改章节主链；
- 完成标志：形成可执行挂接清单与不可破坏约束清单。

### 3.2 第 1 轮：定义全书规划对象与卡槽种子 schema

- 目标：明确全书对象、种子对象、三态流转与字段映射；
- 不做：不开发生成器、不接入 UI；
- 完成标志：Schema 评审通过并形成统一命名映射。

### 3.3 第 2 轮：补齐全书规划生成 / 编辑的最小入口

- 目标：提供最小创建/编辑/读取能力，沉淀规划摘要；
- 不做：不自动建卡、不自动入史；
- 完成标志：全书规划对象可稳定落地并可被读取。

### 3.4 第 3 轮：补齐从规划生成卡槽候选与作者确认

- 目标：完成四类卡槽候选生成与作者确认最小闭环；
- 不做：不做全文抽取、不做复杂冲突自动合并；
- 完成标志：候选卡与确认卡状态可追踪、可审计。

### 3.5 第 4 轮：将规划摘要与确认卡槽接入章节蓝图输入

- 目标：让 ChapterBlueprint 在生成时引用上游规划资产；
- 不做：不重写蓝图生成主流程，不改变 Gate/Publish 语义；
- 完成标志：蓝图输入明确包含全书规划摘要与确认卡槽引用。

## 4. 每轮目标

每轮均需满足以下共性目标：

1. 不破坏 Canon / Story State / ChangeSet 中枢语义；
2. 不破坏既有章节主链运行稳定性；
3. 每轮输出可被下一轮直接复用；
4. 每轮结束均可独立验收并记录风险。

## 5. 每轮不做事项

跨轮统一不做事项：

- 不做自动 Canon 写入；
- 不做自动 ChangeSet 生成与审批；
- 不做自动全文抽取建卡；
- 不做复杂知识图谱；
- 不做章节主链重构；
- 不继续扩新卡型。

## 6. 每轮完成标志

- 第 0 轮：审计报告签收；
- 第 1 轮：对象与 Schema 评审通过；
- 第 2 轮：全书规划入口联调通过；
- 第 3 轮：候选生成与作者确认流程回归通过；
- 第 4 轮：蓝图输入挂接验收通过。

## 7. 轮次依赖关系

- 第 0 轮是第 1~4 轮前提；
- 第 1 轮完成后，方可进入第 2/3 轮实现；
- 第 2 与第 3 轮完成后，方可进入第 4 轮挂接验收；
- 不允许跳过前置轮次直接并行开发。

## 8. 不应出现的执行偏差

1. 未完成对象审计即启动自动生成开发；
2. 将候选卡误当 Canon 正式卡直接写入；
3. 以“效率”为由绕过 ChangeSet；
4. 在 MVP 阶段提前扩展复杂图谱与新卡型；
5. 在 00~04 未评审通过前提前进入实现工单。

## 9. 与未来 05 执行任务单的关系

本文件用于阶段规划，不是执行工单。仅当 00~04 决策链评审通过、并明确进入实现阶段后，才可编制 `05_Codex执行任务单_全书级规划与卡槽种子生成.md`。

## 10. 最终确认结论

当前阶段应先完成“全书级规划与卡槽种子生成”立项收口，并以 00~04 作为唯一上游输入。完成评审前，不进入实现开发。
